## Commits (in order)

### common: add load_fits_or_none, speed up removebadpix, misc fixes
`support_functions.py`:
  - `load_fits_or_none(path)`: loads FITS primary HDU or returns None.
    Used by qacits and fnf for optional calibration files.
  - `removebadpix`: rewrote to use `skimage.util.view_as_windows` so cost
    scales with number of bad pixels, not image size.
  - `validate_config`: raises `ValueError` instead of `sys.exit(0)`.
  - Replaced deprecated `np.float` with `float`.

`classes.py`:
  - Fixed typo: `self.control_even_modes = control_odd_modes` ->
    `self.control_even_modes = control_even_modes`.

### qacits: pending edits and README
All qacits working-tree edits committed: GUI, plotter, config, funcs, etc.
`README.md` added (new file explaining quad-cell method, config parameters,
hitchhiker mode, simulation, tuning guide).

### fnf: add CAMERA CALIBRATION section, dm command boost
Replaced `N images averaged` / `auto_background` with `dm command boost`
and a `CAMERA CALIBRATION` section wired through `load_fits_or_none`.
Updated `FF_software.ini`, `FF_software.spec`, `FF_software_sim.ini`,
`gui_helper.py`, `FF_GUI.py` (file-picker widget), `run.py`.

### nirc2 changes (Mahawa)
  - New `qacits/qacits_hardware.py`: `NIRC2Alias` (camera wrapper) and
    `K2AOAlias` (FSM tip-tilt wrapper using `aoscripts.ao_systems.k2ao`).
  - `qacits_gui_helper.py`: `load_instruments('NIRC2')` returns the aliases.

### qacits: unify AO interface around offset_tiptilt / zero_tiptilt (Option A)
Replaced `get_dm_data / set_dm_data + generate_tip_tilt` flow with a
common `offset_tiptilt(x, y)` / `zero_tiptilt()` interface:
  - `FakeAODMSystem`: implemented the existing stub + added `zero_tiptilt`.
  - `run_qacits.py`: rotation/flip applied inline via `dm.rotate_flip_tt`
    before calling `AOSystem.offset_tiptilt(x, y)`. Removed `import dm`
    (later re-added when rotate_flip_tt moved to dm.py).
  - `qacits_gui_helper.py`: tightened to explicit `elif 'NIRC2'` with
    `ValueError` for unknown names; fixed import to package-relative.

### dm: add rotate_flip_tt helper
Moved the 2-vector rotation/flip transform from a nested function in
`run_qacits` to `common/dm.py` as `rotate_flip_tt` — the natural sibling
of `generate_tip_tilt`. Purely additive; no other callers affected.

### qacits: add interactive NIRC2 image grabber
New `qacits/nirc2_image_grabber.py`: run `python -i` on Keck, call
`save_image('/dir')` to drop a NIRC2 FITS into a directory for
hitchhiker-mode testing.

### san: calibration GUI + locate_badpix plot polish
New `san/detector_calibration_GUI.py`:
  - PyQt5 tool with Sim / NIRC2 selector, output-dir browser,
    # images spinbox, three buttons (Take BGD / FLAT / BADPIX).
  - Confirmation dialog before each acquisition.
  - Acquisition runs in QThread with per-frame progress.
  - BGD: saves median as `masterbgd.fits`.
  - FLAT: saves mean-normalized median as `masterflat.fits`.
  - BADPIX: runs `locate_badpix(median, sigmaclip=3, plot=True)`,
    saves as `badpix.fits`.
  - Recommendations table printed in the GUI.
  - `matplotlib.use('Qt5Agg')` forced so plots coexist with Qt.

`support_functions.py`:
  - `locate_badpix` plot: switched from semilogy to log-y + symlog-x.

`sn_config.ini`:
  - Blanked `output_directory` under `SIMULATION.CAMERA_PARAMS` so
    FakeDetector no longer tries to save to `/Users/mbottom/...` on
    non-Mac hosts. Callers that need saves pass their own path.

### sim: print on every FakeDetector acquisition
FakeDetector now prints `"Sim image acquired at HH:MM:SS shape=..."`.
Calibration GUI also prints `"Taking image N/total for <kind>"`.

### locate_badpix: force float arithmetic in percentage calc
`100 * np.sum(bpmask)` overflowed when bpmask was uint8 (numpy small-int
dtype from `np.round(bool) + np.round(bool)`). Cast to float first.

### calibration GUI: normalize flat; tweak recommendations table
Flat is now mean-normalized before saving. Table text cleaned up,
font bumped to 11pt Courier.

## Known remaining issues

1. **`bgddir` in `sn_config.ini`** still has the macOS path. Affects
   san calibration scripts on Keck if anyone runs them (not used by
   qacits or fnf).

2. **AO on Keck is broken** (hardware issue). We confirmed NIRC2 camera
   readout works. AO / FSM testing deferred.

3. **`qacits_camera_image_generator.py`** still references the old
   `dm.generate_tip_tilt` for its sim setup. Works but has not been
   updated to use `offset_tiptilt` since it directly builds the optical
   model rather than going through AOSystem.

4. **Hardcoded macOS paths** in `qacits_config.ini` (`imagedir`) are
   site-specific. Each user must edit these for their machine.

5. **`equalize_image` scalar-median fallback** is silent. Could add a
   one-time warning when no background file is provided.
